# 个人知识库 (RAG)

你每天都在阅读文章、浏览推文、观看视频，但总是找不到上周看到的那条内容。书签越堆越多，最终变得毫无用处。

这个工作流将你保存的所有内容构建为一个可搜索的知识库：

• 将任意 URL 发送到 Telegram 或 Slack，系统会自动抓取并收录内容（文章、推文、YouTube 字幕、PDF）
• 对所有已保存内容进行语义搜索："我保存过哪些关于 agent 记忆的内容？" 返回带有来源的排序结果
• 可为其他工作流提供数据支持——例如视频创意管线在制作研究卡片时会自动查询知识库获取相关已保存内容

## 所需技能

- [knowledge-base](https://clawhub.ai) 技能（或使用嵌入向量自建 RAG）
- `web_fetch`（内置）
- 用于内容收录的 Telegram 话题或 Slack 频道

## 配置方法

1. 从 ClawdHub 安装 knowledge-base 技能。
2. 在 Telegram 中创建一个名为 "knowledge-base" 的话题（或使用 Slack 频道）。
3. 向 OpenClaw 发送以下提示词：
```text
当我在 "knowledge-base" 话题中发送 URL 时：
1. 抓取内容（文章、推文、YouTube 字幕、PDF）
2. 将内容连同元数据（标题、URL、日期、类型）收录到知识库中
3. 回复确认：已收录的内容和分块数量

当我在该话题中提问时：
1. 对知识库进行语义搜索
2. 返回带有来源和相关摘录的顶部结果
3. 如果没有良好匹配，请告知我

此外：当其他工作流需要研究资料时（例如视频创意、会议准备），自动查询知识库获取相关的已保存内容。
```

4. 测试：发送几个 URL，然后提问"我有哪些关于 LLM 记忆的内容？"
