# 认证门户监控

许多重要系统仍然隐藏在登录墙后面：计费仪表盘、合作伙伴门户、物流控制台、内部管理面板和供应商后台。这些系统通常没有可用的 API，简单的爬虫无法处理，而且需要真正的浏览器会话才能查看变化。

此工作流为 OpenClaw 提供了一条基于浏览器的门户监控路径。智能体可以自动打开网站、登录、检查你关注的页面、提取重要数据，并按计划发送带截图或 PDF 的摘要报告。

## 痛点

你需要从以下类型的 Web 应用获取更新：

- 需要登录认证
- 大量客户端渲染
- 对简单的 HTTP 请求不友好
- 每天手动检查过于繁琐

示例：

- 客户或供应商仪表盘
- 广告平台和分析门户
- 没有 API 的内部工具
- 订单管理和客服控制台
- 需要截图留证的财务、合规或计费门户

## 功能说明

OpenClaw 使用浏览器工作流来：

- 为目标门户打开真实的浏览器会话
- 登录并导航多步骤流程
- 提取你关注的指标、告警或表格数据
- 截取屏幕截图或导出 PDF 作为审计凭证
- 按计划通过 Telegram、Slack、Discord、邮件或其他渠道发送结果

当你需要摘要和凭证时，此方案特别有用。摘要告诉你什么发生了变化；截图或 PDF 让你无需亲自打开网站就能快速验证。

## 所需技能

- [steel-browser](https://github.com/steel-dev/cli/tree/main/skills/steel-browser) — 可复用的浏览器技能，支持会话级导航、数据提取、截图和 PDF 导出
- [Steel CLI](https://github.com/steel-dev/cli) — 可选的终端优先方式，实现相同的浏览器工作流
- OpenClaw 定时任务 — 用于定期检查和报告

## 设置方法

1. 将 `steel-browser` 技能添加到你的 OpenClaw 工作区。

2. 给 OpenClaw 一条常驻指令，如下所示：

```text
你帮我监控需要登录认证的 Web 门户。

当收到门户监控任务时：
- 使用 steel-browser 工作流
- 逐步登录并导航网站
- 仅提取我要求的字段
- 当页面重要或发生变化时，截取屏幕截图或导出 PDF
- 先返回简洁摘要，然后附上凭证
- 如果登录失败或页面结构发生变化，准确报告失败位置

不要猜测数值。如果字段缺失或不明确，请明确说明。
```

3. 创建一个循环任务或定时任务。示例指令：

```text
每个工作日早上 8:30，打开 Acme 合作伙伴门户并检查：
- 新订单
- 失败订单
- 未读支持工单升级
- 账户余额

在 Telegram 中发送简短摘要。
如果存在失败订单或工单升级，请附上相关页面的截图。
如果余额与昨天相比有变化，将计费页面导出为 PDF。
```

## 使用场景示例

### 计费仪表盘检查

```text
每天早上，登录我们的计费门户并告诉我：
- 昨天的支出
- 本月累计支出
- 是否有新的支付失败

如果存在支付失败，发送失败记录表的截图。
```

### 电商卖家运营

```text
每天两次，检查卖家门户：
- 新订单
- 退款请求
- 违规通知

如果存在违规通知，发送具体文字内容加截图。
```

### 内部管理 QA

```text
每天上午 9 点和下午 4 点，登录我们的预发布管理面板，
确认队列长度、待处理任务和失败任务，然后发送摘要。
如果失败任务数不为零，附上任务页面的截图。
```

## 关键经验

- 此方案最适合 API 缺失、不完整或比浏览器自动化更难维护的网站。
- 截图和 PDF 使工作流可审计。这对财务、合规和运营审查很重要。
- 保持提取规则聚焦。只要求少量特定字段，而不是"总结页面上所有内容"。
- 对于你知道每天都会访问的门户，定时监控优于被动检查。
- 如果门户使用 MFA 或定期重新认证，请在智能体指令中记录预期的登录行为，以便更容易诊断故障。
