# 播客制作流水线

你有一个播客的想法，甚至有一集 backlog 话题。但是在研究嘉宾、撰写大纲、起草开场白、生成节目笔记和写社交媒体推广帖子之间——制作开销扼杀了你的势头。如果你交出一个话题就能得到一个完整的制作包呢？

这个用例将智能体串联起来，处理从话题到发布就绪资产的整个播客制作工作流程。

## 痛点

独立播客主持人和小团队在制作上花的时间比实际录制还多。研究需要数小时，节目笔记是事后才想到的，社交媒体推广是最先被跳过的部分。创意部分——对话——可能只占总工作量的 30%。这个智能体处理另外 70% 的工作。

## 功能概述

- **单集研究** ——给定一个话题或嘉宾姓名，汇编背景研究、谈话要点和建议问题
- **大纲和脚本** ——生成包含开场白脚本、环节过渡和结束语的结构化单集大纲
- **节目笔记** ——录制后，将转录稿处理成带时间戳的节目笔记，包含提到的所有内容的链接
- **社交媒体套件** ——为 X、LinkedIn 和 Instagram 创建推广帖子，包含亮点和摘录
- **单集描述** ——为 Spotify、Apple Podcasts 和 YouTube 编写 SEO 优化的单集描述

## 所需技能

- 网络搜索/研究技能（用于嘉宾研究和话题深度挖掘）
- 文件系统访问（用于阅读转录稿和写入输出文件）
- Slack、Discord 或 Telegram 集成（用于传递资产）
- 可选：`sessions_spawn` 用于并行运行研究和写作智能体
- 可选：RSS 源技能（用于监控竞品播客）

## 如何设置

1. 录制前 —— 生成研究和大纲：
```text
我正在录制一个关于 [话题] 的播客单集。嘉宾是 [姓名]。

请：
1. 研究嘉宾——他们的背景、近期工作、尖锐观点，以及
   他们在公开场合说的任何有争议或有趣的内容。
2. 研究话题——关键趋势、最新新闻、常见误解，
   以及观众可能已经知道什么 vs 会让他们感到惊喜的内容。
3. 生成单集大纲：
   - 冷开场钩子（1-2句话抓住注意力）
   - 开场白脚本（30秒，轻松语气）
   - 5-7个采访问题，从简单/建立融洽关系到深入/引人深思排序
   - 2-3个"备用"问题，以防对话停滞
   - 结尾环节，带有行动号召

将所有内容保存到 ~/podcast/episodes/[集数]/prep/
```

2. 录制后 —— 生成节目笔记和推广内容：
```text
这是第 [集数] 期的转录稿：[粘贴或指向文件]

请：
1. 写带时间戳的节目笔记——每个主要话题转换都带有时间戳
   和一行摘要。包含提到的任何内容的链接（工具、书籍、
   文章、人物）。
2. 写单集描述（最多 200 字），针对播客
   搜索优化。自然包含 3-5 个相关关键词。
3. 创建社交媒体帖子：
   - X/Twitter：3 条推文——一条摘录，一条关键见解，一条
     引发讨论的问题。每条不超过 280 字符。
   - LinkedIn：1 篇帖子，专业语气，100-150 字。
   - Instagram 描述：1 篇带表情符号的帖子，轻松语气，包含相关标签。
4. 提取"亮点"列表——3 个最有趣/令人惊讶的时刻
   及其时间戳。

将所有内容保存到 ~/podcast/episodes/[集数]/publish/
```

3. 可选 —— 竞品监控：
```text
每天监控这些播客 RSS 源：
- [源 URL 1]
- [源 URL 2]

当出现涵盖与我播客相关话题的新单集时，
向我发送 Telegram 消息，包含：
- 单集标题和链接
- 一句话摘要
- 这是否是我应该回应或从我的角度覆盖的内容
```

## 核心洞察

- **录制前的研究** 是价值最大的部分。带着深入的嘉宾研究进入采访会让对话显著改善——这是你在后期制作中无法伪造的。
- 带时间戳的节目笔记是听众留存的重要工具。大多数播客主持人会跳过它们，因为很繁琐。这个智能体让它们变得毫不费力。
- 社交媒体套件节省了最多的**重复**时间。每一单集都需要推广，而且总是相同结构——非常适合自动化。
- 如果你想将播客内容重新制作成博客文章、通讯或视频片段，可以很好地搭配 [多智能体内容工厂](content-factory.md)。
