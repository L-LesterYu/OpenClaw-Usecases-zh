# B2B AI 销售开发代表 (SDR)

出口公司和制造企业每天在重复性销售任务上耗费大量时间：调研潜在客户、撰写开发邮件、WhatsApp 跟进、资质评估以及更新 CRM 记录。大多数小型贸易公司负担不起专职销售团队，创始人只能一个人硬扛——而且做得并不好。

本用例将 OpenClaw 变为 AI 驱动的 SDR，在 WhatsApp、Telegram 和邮件上全渠道运行 B2B 销售管道。7×24 小时在线，搭载 7 层上下文系统，了解你的公司、产品、理想客户画像和销售话术。

## 痛点

- B2B 出口销售劳动密集：调研 → 触达 → 跟进 → 资质评估 → 报价 → 谈判——一个人每天最多管理约 20 条线索
- 小型贸易企业（工厂、出口商）雇不起 SDR 团队——创始人/老板一边管运营一边做销售
- 通用聊天机器人不懂 B2B 语境：产品规格、起订量（MOQ）、贸易术语（FOB/CIF）、付款条件（T/T、L/C）
- 销售跟进容易遗漏——买家凌晨 2 点在 WhatsApp 上提问，等你早上回复时人家已经去找竞争对手了
- 缺乏 CRM 纪律——对话分散在 WhatsApp、邮件和微信中，没有统一的销售管道

## 功能特性

- **7 层上下文系统**：IDENTITY.md（公司信息）、SOUL.md（人设/规则）、AGENTS.md（10 阶段销售工作流）、USER.md（理想客户画像 + 产品目录）、HEARTBEAT.md（管道巡检）、MEMORY.md（三引擎记忆）、TOOLS.md（CRM + 渠道配置）
- **全管道自动化**：线索捕获 → 资质评估 → 需求分析 → 产品匹配 → 报价 → 谈判 → 跟进 → 成交
- **多渠道**：同时在 WhatsApp、Telegram 和邮件上响应，跨渠道保持上下文一致
- **10 个定时任务**：晨间管道复盘、线索评分更新、跟进提醒、周报、停滞交易告警
- **智能资质评估**：根据国家、行业、订单规模和互动情况为线索打分，让智能体将精力集中在高价值潜在客户上
- **文化感知**：根据不同市场调整沟通风格（中东偏正式、欧洲偏直接、亚洲偏关系导向）
- **人工升级**：知道何时转交给企业主——大额订单、定制需求，或买家坚持要与真人沟通

## 所需技能

- 已配置 WhatsApp 频道的 OpenClaw（或 Telegram/邮件）
- ClawHub 上的 b2b-sdr-agent 技能，或完整模板仓库
- 你的产品目录和公司信息（填入 7 个 Markdown 文件）

## 部署方法

### 1. 安装

```bash
# 方式 A：通过 ClawHub
clawhub install b2b-sdr-agent

# 方式 B：完整仓库
git clone https://github.com/iPythoning/b2b-sdr-agent-template.git
```

### 2. 自定义 7 层上下文

每一层都是一个 Markdown 文件，根据你的业务进行编辑：

```bash
vim IDENTITY.md   # 公司名称、角色、主营业务
vim USER.md       # 产品、定价、理想客户画像、竞争对手
vim SOUL.md       # 沟通规则、语气、边界
vim AGENTS.md     # 10 阶段销售工作流
vim HEARTBEAT.md  # 定时管道检查
vim MEMORY.md     # 智能体如何记忆过往互动
vim TOOLS.md      # CRM 设置、频道配置
```

将所有 `{{占位符}}` 替换为你的实际信息。

### 3. 连接渠道

通过 OpenClaw 的频道系统添加 WhatsApp。智能体将使用你的销售上下文自动响应 incoming 消息。

### 4. 用真实线索测试

在 WhatsApp 上发送测试消息："你好，我对你们面向非洲市场的产品感兴趣。你们的起订量是多少？"

智能体应该回复产品详情，提出资质评估问题，并将线索推进到销售管道中。

## 核心洞察

- **上下文分层 > 提示词工程**：与其用一个超长提示词，不如将上下文拆分为 7 个专注的 Markdown 文件（身份、人设、工作流、记忆、工具），这能让智能体在人设保持和销售流程遵循方面表现显著提升。每个文件不超过 2000 token，但合在一起提供了深厚的业务上下文。
- **定时任务是 SDR 的纪律保障**：真正的销售成功来自持续跟进，而非惊艳的首条消息。10 个定时任务（晨间管道复盘、停滞交易告警、周报）执行了人类容易遗忘的纪律。
- **B2B 销售需要记忆**：与客服不同，B2B 销售周期跨越数周甚至数月。三引擎记忆系统（对话记忆、交易记忆、关系记忆）让智能体能自然引用过往互动——"上个月您提到正在拓展肯尼亚市场，那些计划确定了吗？"
- **多渠道是基本功**：不同地区的 B2B 买家偏好不同的渠道。中东买家用 WhatsApp，欧洲买家偏好邮件，东南亚买家用 Telegram。智能体必须在所有渠道上维护统一的上下文。

## 灵感来源

基于与中国出口企业（制造商、贸易公司）向 50+ 个国家销售的真实部署经验构建。7 层上下文架构通过在实际销售对话中的迭代测试逐步完善——每一层都是在智能体缺少该层时持续失败于特定任务后才添加的。
