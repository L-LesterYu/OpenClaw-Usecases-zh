# 自动化会议纪要与行动项

你刚结束一个 45 分钟的团队会议。现在你需要撰写会议摘要、整理行动项，并把它们分发到 Jira、Linear 或 Todoist——全靠手动操作。等你忙完，下一场会议又要开始了。如果你的 Agent 在会议纪要落地的瞬间就自动搞定这一切呢？

本用例将任意会议纪要转化为结构化笔记，并自动在项目管理工具中创建任务。

## 痛点

会议纪要繁琐但至关重要。大多数人要么跳过（导致上下文丢失），要么花 20 多分钟来撰写。行动项被遗忘，因为它们只存在于某人的脑海中，或者淹没在聊天记录里。这个 Agent 消除了"我们讨论过了"和"已追踪并分派"之间的鸿沟。

## 功能说明

- **监听**新的会议纪要（通过 Otter.ai 导出、Google Meet 纪要、Zoom 录制摘要，或直接粘贴到聊天中）
- **提取**关键决策、讨论主题和行动项（含负责人和截止日期）
- **创建任务**到 Jira、Linear、Todoist 或 Notion——分配给正确的人员并附带会议上下文
- **发布摘要**到 Slack 或 Discord，让整个团队都有记录
- **跟踪提醒**——可选地通过定时提醒在截止日期前通知负责人

## 所需技能

- Jira、Linear、Todoist 或 Notion 集成（用于创建任务）
- Slack 或 Discord 集成（用于发布摘要）
- 文件系统访问（用于读取纪要文件）
- 定时任务 / cron（用于跟踪提醒）
- 可选：Otter.ai、Fireflies.ai 或 Google Meet API，用于自动获取纪要

## 如何配置

1. 选择你的纪要来源。最简单的方式是直接将纪要粘贴到聊天中。如需自动化，可设置文件夹监听或 API 集成。

2. 向 OpenClaw 发送提示：
```text
我刚结束一个会议。这是会议纪要：

[粘贴纪要或指向文件]

请：
1. 撰写一份简明摘要（最多 5 个要点），涵盖关键决策和讨论主题。
2. 提取所有行动项。对每一项，识别：
   - 需要做什么
   - 谁负责（将姓名匹配到我的团队）
   - 截止日期（如有提及，否则标注为"TBD"）
3. 为每个行动项创建一个 Jira 任务，分配给对应负责人。
4. 将完整摘要发布到 Slack 的 #meeting-notes 频道。
```

3. 如需全自动流水线（纪要文件夹监听）：
```text
设置一个定时任务：每 30 分钟检查 ~/meeting-transcripts/ 目录中的
新 .txt 或 .vtt 文件。发现新文件时：

1. 将纪要解析为带行动项的结构化摘要。
2. 在 Linear 中为每个行动项创建任务。
3. 将摘要发布到 Slack 的 #team-updates 频道。
4. 将已处理的文件移动到 ~/meeting-transcripts/processed/ 目录。

对于每个有截止日期的行动项，设置提醒在截止日期前一天
在 Slack 中通知负责人。
```

4. 自定义输出格式：
```text
撰写会议摘要时，始终使用以下结构：
- 顶部标注 **日期与参会人员**
- **关键决策** — 编号列表，列出已决定的事项
- **行动项** — 表格，列名为：任务、负责人、截止日期、状态
- **待解决问题** — 任何需要后续跟进的未决事项
```

## 关键洞察

- 真正的价值不在于摘要本身——而在于**自动创建任务**。不会变成已追踪任务的会议纪要，只是"文档表演"。
- 将此用例与 [Todoist 任务管理器](todoist-task-manager.md) 结合使用，可全面了解 Agent 创建的任务。
- 来自 Zoom 或 Google Meet 的 VTT/SRT 字幕文件非常适合作为输入——它们包含时间戳，有助于 Agent 将发言归因到对应的发言人。
- 从简单开始（粘贴纪要，获取摘要），然后逐步自动化。在验证输出质量之前，不要过度工程化流水线。
