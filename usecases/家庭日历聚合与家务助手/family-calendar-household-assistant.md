# 家庭日历聚合与家务助手

现代家庭往往需要管理五个甚至更多的日历——工作、个人、共享家庭、孩子学校、课外活动——而且分散在不同的平台和格式中。重要的日程安排经常因为缺乏统一的视图而被遗漏。与此同时，家务协调（购物清单、食品柜库存、预约排期）只能通过散落的短信来完成，信息很快就被淹没。

本用例将 OpenClaw 变为一个随时在线的家庭协调员：聚合多个日历生成晨间简报、监控消息中的可操作事项，并通过共享聊天界面管理家庭后勤。

## 痛点

- **日历碎片化**：工作日历有安全限制无法共享。学校日历以 PDF 或手写网页形式提供。夏令营安排藏在邮件里。每天早上逐一查看每个日历不可持续——"把事件跨日历复制过去确实有效，直到我忘了，然后就漏掉一个。"
- **家务协调开销**："家里还有多少牛奶？"需要先去冰箱看看，再去地下室储藏室看看，然后发短信回复。把这种情况乘以一周的购物次数。
- **遗漏预约**：预约确认通过短信到达后就静静地待在那里——没有创建日历事件、没有预留通勤时间、没有提醒。

## 功能概述

- **晨间简报**：将所有家庭日历聚合为每日摘要，通过你偏好的渠道推送
- **被动消息监控**：监控 iMessage/短信对话，自动检测到预约（牙医确认、会议安排等）时创建日历事件
- **通勤时间缓冲**：在检测到的预约前后自动添加通勤时间段
- **家庭库存管理**：维护食品柜/冰箱物品的实时库存，任何一方都可以随时查询
- **购物协调**：跨菜谱去重食材、追踪即将耗尽的物品、生成购物清单
- **拍照输入**：拍摄学校日历或冷冻室内容的照片，智能体会将其处理为结构化数据

## 所需技能

- 日历 API 访问（Google Calendar、通过 `ical` 访问 Apple Calendar）
- `imessage` 技能用于消息监控（仅限 macOS）
- Telegram 或 Slack 作为共享家庭聊天界面
- 文件系统访问用于库存追踪
- 摄像头/照片处理能力，用于物理日历的 OCR 识别

## 如何设置

### 1. 日历聚合

配置 OpenClaw 从所有家庭日历来源拉取数据：

```text
## 日历来源

晨间简报（上午 8:00）：

1. 获取我的 Google 工作日历（只读 OAuth）
2. 获取共享的家庭 Google 日历
3. 获取伴侣的日历（共享视图）
4. 检查 ~/Documents/school-calendars/ 中的新 PDF → OCR 提取事件
5. 检查最近的邮件中是否有日历附件或事件邀请

汇总为统一简报：
- 今日事件（所有日历，按来源着色标注）
- 未来 3 天冲突预警
- 自昨天起新增的事件
- 户外活动的天气信息

通过 Telegram/Slack 家庭频道推送。
```

### 2. 被动消息监控

这是核心差异化功能——智能体被动观察，在识别到可操作内容时自动行动：

```text
## 消息监控（HEARTBEAT.md）

每 15 分钟：
1. 检查所有对话中的新 iMessage
2. 检测类似预约的模式：
   - "您的预约已确认，时间为..."
   - "我们可以在 [日期] [时间] 见面吗？"
   - "训练改到周六下午 3 点"
3. 检测到时：
   - 创建包含详细信息的日历事件
   - 在前后各添加 30 分钟的通勤缓冲时间
   - 发送确认到家庭 Telegram："已创建：牙医预约，周二下午 2 点。已添加通勤时间 1:30-2:00 和 3:00-3:30。"
   - 如涉及伴侣，添加邀请
4. 检测承诺/约定模式：
   - "我会在周五之前发给你"
   - "下周一起吃饭吧"
   → 创建日历占位或提醒
```

### 3. 家庭库存管理

```text
## 食品柜追踪

维护 ~/household/inventory.json，包含：
- 物品名称、数量、位置（冰箱/食品柜/地下室）
- 最后更新时间戳
- 低库存阈值

更新方式：
- 拍照：用户发送冰箱/食品柜照片 → 视觉模型提取物品
- 文字："鸡蛋没了" / "买了 2 加仑牛奶"
- 收据：拍摄购物小票照片 → 更新库存

查询：任何一方都可以通过 Telegram 提问：
- "家里有黄油吗？" → 查询库存，回复位置和数量
- "什么东西快用完了？" → 列出低于阈值的物品
- "生成购物清单" → 汇总低库存物品 + 所需菜谱食材
```

## 核心洞察

- **被动 > 主动**：最大的突破是智能体在未被要求时就采取行动。在短信中检测到预约并自动创建带有通勤缓冲的日历事件——"我没有让它这么做，它就知道这是我想要的。"
- **Mac Mini 是最佳选择**：这个用例在家庭 Mac Mini 上运行有巨大优势——iMessage 集成、Apple Calendar、以及始终在线的可用性
- **先从只读开始**：先从日历读取和消息监控开始，再启用写入操作（创建事件、发送消息）
- **共享 Telegram 频道**：让双方都能看到智能体在做什么——建立信任并尽早发现错误
- **拍照输入被低估了**：拍一张学校日历 PDF 或冷冻室内容的照片比打字快得多——而且视觉模型处理得很好

## 灵感来源

本用例结合了多种社区模式：

- **日历聚合**：HN 用户 `angiolillo` 在 [Hacker News 讨论](https://news.ycombinator.com/item?id=46872465)中描述了每天早上分别查看工作、个人、家庭和孩子学校日历的痛苦。
- **被动消息监控**：[Sparkry AI](https://sparkryai.substack.com/p/24-hours-with-openclaw-the-ai-setup) 记录了这一功能——当妻子收到牙医预约短信时，OpenClaw 自动创建了带有 30 分钟通勤缓冲的日历事件，无需任何要求。[OpenClaw Showcase](https://openclaw.ai/showcase) 中 `@theaaron` 也确认了这一点，称基于聊天的日历管理是"我体验过的 LLM 最佳用途之一"。
- **家务协调**：Brandon Wang 的 [Clawdbot "Linguini"](https://brandon.wang/2026/clawdbot) 运行在家里的 Mac Mini 上——处理短信跟进、从照片创建日历事件、追踪 Airbnb 价格、处理冷冻室库存照片，并通过 iMessage 和 Slack 协调家庭后勤。
- **食品柜追踪**：多位 HN 用户讨论了维护家庭库存的价值和挑战，`dns_snek` 表示："我放下东西 5 秒钟就忘了放在哪……这对我来说确实是个大问题，因为我会让食物过期。"
