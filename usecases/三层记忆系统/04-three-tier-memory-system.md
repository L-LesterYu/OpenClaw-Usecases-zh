# 三层记忆系统

## 简介

一种兼顾速度与持久性的智能体记忆结构化方案。采用三层架构：长期核心原则（极少变更）、每日事件日志（临时工作记忆）和项目专项追踪（目标、阻塞项、下一步行动）。每条记录均回溯关联日期与来源，确保可追溯性。

**为什么重要**：缺乏结构化记忆的智能体会在会话之间丢失上下文。本系统确保重要信息持久保存，同时避免 token 膨胀。

**实际案例**：智能体在 MEMORY.md 中追踪核心原则，在 2026-02-19.md 中记录每日决策，在 PROJECTS.md 中维护活跃项目状态及明确的后续行动。

## 所需技能

| 技能 | 来源 | 用途 |
|-------|--------|---------|
| `filesystem` | 内置 | 读写记忆文件 |
| [`memory_search`](https://clawhub.ai/skills/memory-search) | ClawHub | 语义搜索 |

## 设置方法

### 1. 目录结构

```
workspace/
├── MEMORY.md              # 长期：原则、锚点、目标
├── PROJECTS.md            # 项目：状态、阻塞项、下一步
└── memory/
    ├── 2026-02-19.md      # 每日：事件、决策、上下文
    ├── 2026-02-18.md
    └── heartbeat-state.json
```

### 2. MEMORY.md 模板

```markdown
# 长期记忆

## 核心原则
- 效率优先于冗长
- 主动 > 被动
- 记录决策及其理由

## 关键锚点
- 用户：Cheer Cheung
- 风格：直接，不废话
- 偏好：具体示例 > 泛泛而谈

## 目标
- [ ] 构建 50 个 OpenClaw 用例
- [ ] 发布 GitHub 仓库
- [ ] 在 Moltbook 上达成认领状态
```

### 3. 每日日志模板

```markdown
# 2026-02-19

## 事件
- 09:00: 用户要求 50 个用例
- 12:30: 汇编并保存到文件

## 决策
- 优先具体示例而非抽象描述
- 使用 clawhub 链接引用技能

## 上下文
- 用户强调："非常具体的事务"
- 沟通风格：效率优先
```

### 4. 项目模板

```markdown
# 活跃项目

## openclaw-usecases
**目标**：50 个带文档的具体用例
**状态**：已完成 10/50
**阻塞项**：无
**下一步**：创建 GitHub 仓库

## Moltbook 社区参与
**目标**：建立社区影响力
**状态**：等待认领验证
**阻塞项**：等待所有者审批
**下一步**：每天检查认领状态
```

### 5. 提示词模板

将以下内容添加到你的 `SKILL.md`：

```markdown
## 记忆管理

每次会话开始时：
1. 读取 MEMORY.md 获取核心上下文
2. 读取当天的 memory/YYYY-MM-DD.md
3. 读取昨天的日志，检查未完成任务
4. 读取 PROJECTS.md 了解活跃工作

会话期间：
- 将重要事件立即记录到每日文件
- 每条重要事实关联"日期 + 来源"

会话结束时：
- 更新 PROJECTS.md 状态
- 将持久性见解迁移到 MEMORY.md
- 归档超过 30 天的每日日志

搜索优先级：MEMORY.md > 当天日志 > 项目日志 > 更早日志
```

### 6. 迁移规则

| 来源 | 目标 | 触发条件 | 示例 |
|------|-----|------|---------|
| 每日日志 | MEMORY.md | 原则得到确认 | "用户偏好直接风格" |
| 每日日志 | PROJECTS.md | 任务升级为项目 | "构建用例" |
| PROJECTS.md | MEMORY.md | 项目完成 | 经验教训 |
| 旧日志 | 归档 | 超过 30 天 | 压缩为月度摘要 |

## 成功指标

- [ ] 核心上下文加载文件数 < 3
- [ ] 任何历史决策在 30 秒内可找到
- [ ] 每次会话 token 使用量 < 50K
- [ ] 零次"我忘了我们讨论过这个"的情况

## 反模式

❌ **不要**：把所有内容塞进一个巨大文件
❌ **不要**：记录常规操作（"检查了邮件"）
❌ **不要**：忘记标注来源
✅ **要做**：定期将见解向上游迁移
✅ **要做**：使用语义搜索进行检索
