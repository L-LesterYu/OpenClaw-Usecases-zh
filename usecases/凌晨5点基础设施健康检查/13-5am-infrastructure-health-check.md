# 凌晨5点基础设施健康检查

## 简介

清晨服务器健康监控：运行时间检查、磁盘空间验证、资源利用率追踪和备份校验。在人类开始工作前主动发现潜在问题。

**为什么重要**：早上9点才发现的基础设施问题会浪费数小时的生产力。提前检测可以在造成影响之前完成修复。

**真实案例**：智能体在凌晨5点检测到磁盘使用率达95%，立即告警通知用户，清理脚本自动运行，在上班前化解了一场危机。

## 所需技能

| 技能 | 来源 | 用途 |
|-------|--------|---------|
| `system` | 内置 | 资源监控 |
| `telegram` | 内置 | 告警通知 |
| `cron` | 内置 | 定时自动化 |

## 如何设置

### 1. 健康检查脚本

```bash
#!/bin/bash
# check-health.sh

df -h | grep -E "(9[0-9]|100)%" && alert "Disk critical"
free -m | awk '/Mem:/ {if ($3/$2 > 0.9) print "Memory high"}'
uptime | awk '{if ($3 > 7) print "Reboot suggested"}'
```

### 2. 提示词模板

```markdown
## 凌晨5点基础设施检查

每天 05:00 执行：
1. 检查所有卷的磁盘使用率
2. 验证内存利用率
3. 检查系统负载均值
4. 校验备份完成情况
5. 测试外部网络连通性
6. 如有检查失败：立即告警
7. 否则：静默成功

阈值设置：
- 磁盘 >90%：警告
- 磁盘 >95%：严重
- 内存 >90%：警告
- 负载 >4：需排查
```
