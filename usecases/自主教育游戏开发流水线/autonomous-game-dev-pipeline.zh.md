# 自主教育游戏开发流水线

## 痛点
**起源故事：** 一位"老派游戏爱好者"爸爸想为他的女儿们 Susana（3岁）和即将出生的 Julieta 创建一个安全、无广告、高质量的游戏网站。现有网站充斥着垃圾邮件、攻击性广告和具有欺骗性按钮（暗模式），让他的幼儿感到沮丧。

**挑战：** 构建"清洁、快速、简单"的门户网站很容易。真正的挑战是填充 **40+ 教育游戏**，这些游戏针对特定发展阶段（0-15岁），而无需开发团队。手动开发对于一个独自工作的父母来说太慢了，在几十个游戏之间保持一致性正在变成一场噩梦。

## 它的功能
这个用例定义了一个"游戏开发智能体"，自主管理游戏创建和维护的完整生命周期。工作流程执行 **"Bug 优先"** 策略，智能体必须检查并解决已报告的 Bug，然后才能实现新功能。

**效率：** 这个流水线能够每 **7 分钟产出 1 个新游戏或 Bug 修复**。智能体不知疲倦地遍历 41+ 个计划游戏的待办列表，在新内容创建和纠正先前周期中发现的问题之间交替进行。

当路径清晰时，智能体：
1. **选择**：根据"轮询"策略从队列 (`development-queue.md`) 中识别下一个游戏，平衡各年龄组的内容。
2. **实现**：为游戏编写 HTML5/CSS3/JS 代码，遵循严格的 `game-design-rules.md`（不使用框架，移动优先，离线能力）。
3. **注册**：自动将游戏元数据添加到中央注册表 (`games-list.json`)。
4. **文档**：更新 `CHANGELOG.md` 和 `master-game-plan.md` 状态。
5. **部署**：处理 Git 工作流：获取主干，创建功能分支，使用约定式提交提交更改，然后合并回主干。

## 提示

这个工作流程的核心是给智能体的**系统指令**。这个提示将 LLM 变成一个遵守项目严格结构的有纪律的开发者。

*(**注意：** 生产中实际使用的提示是 **西班牙语** (`es-419`)，以符合项目目标受众（拉丁美洲儿童）和该地区未来贡献者的需要。下面的版本为文档翻译。)*

```text
作为网页游戏开发和儿童 UX 专家。
你的目标是开发生产队列中的下一个游戏。

在开始之前，请阅读并分析以下上下文文件：

1.  Bug 上下文（最高优先级 - 关键）：
    @[bugs/]
    （检查这个文件夹。如果有文件，你的任务是修复**仅第一个文件**（按字母顺序）。忽略其他的 Bug 和游戏队列）。

2.  队列上下文（下一个游戏）：
    @[development-queue.md]
    （在"下一个游戏"部分标记为 [NEXT] 的游戏。仅在没有 Bug 的情况下）。

3.  设计规则（技术标准）：
    @[game-design-rules.md]
    （严格遵循这些规则：纯 HTML/CSS/JS，文件夹结构，移动响应式）

4.  游戏规范（机制和资产）：
    （基于游戏 ID 在 games-backlog/ 中识别相应文件）

5.  中央注册表（集成）：
    @[public/js/games-list.json]
    （你必须在此注册新游戏的文件，以便它出现在主页上）

任务：
0. **Bug 优先！**：如果 `bugs/` 文件夹有内容，你的唯一优先级是修复**按字母顺序排列的第一个 Bug**。创建 `fix/...` 分支，解决**那个** Bug，更新状态，然后合并。**不要尝试一次修复多个 Bug。**
   - 如果没有 Bug，继续下一个游戏：

1. **同步**：`git fetch && git pull origin master`（关键）。
2. 创建新分支：`git checkout -b feature/[game-id]`。
3. 在 'public/games/[game-id]/' 中创建文件夹和文件。
4. 根据待办事项列表和设计规则实现逻辑和设计。
5. 在 'games-list.json' 中注册游戏（关键）。
6. 完成后：
    - 更新 `CHANGELOG.md` 提升版本。
    - 更新 `master-game-plan.md` 和 `development-queue.md`。
    - 记录更改：`git commit -m "feat: add [game-id]"`。
7. **交付**：
    - 推送：`git push origin feature/[game-id]`。
    - 请求合并到主干。
    - 进入主干后，推送更改 (`git push origin master`)。
```

## 所需技能
- **Git**：管理分支、提交和合并。
