# Docker沙盒化多机器人管理

## 概述

通过原生桌面GUI管理多个OpenClaw实例，每个机器人在独立的Docker容器中运行。容器默认以 `--network none` 启动——网络访问需按机器人单独开启。

## 为什么需要？

OpenClaw拥有操作系统级别的访问权限，可以接触到你的邮件、日历、文件和消息平台。直接在主机上运行意味着一旦遭遇提示注入攻击，攻击者就能获取你的整台机器。ClawPier将每个机器人隔离在独立的Docker容器中，默认禁用网络，因此即使智能体被攻破，也无法访问你的主机或互联网。

## 安装设置

### 安装ClawPier

**macOS（Homebrew）：**
```bash
brew tap SebastianElvis/clawpier
brew install --cask clawpier
```

**或直接下载** [GitHub Releases](https://github.com/SebastianElvis/clawpier/releases) —— 提供 macOS（已签名和公证）、Linux（.AppImage、.deb）和 Windows（.exe、.msi）版本。

### 前置条件

- Docker必须已安装并运行
- ClawPier会在首次启动时提示你拉取OpenClaw镜像

## 工作原理

1. **创建机器人** —— 给它起个名字，可选设置工作空间路径
2. **配置** —— 从GUI设置环境变量、资源限制（CPU/内存）、网络模式和端口映射
3. **启动** —— 机器人在自己的Docker容器中运行，默认与主机隔离
4. **交互** —— 使用内置的聊天、终端、日志查看器、文件浏览器或技能浏览器
5. **监控** —— 实时查看每个机器人的CPU、内存和网络I/O统计

### 核心安全特性

- **网络隔离** —— 每个机器人的容器启动时无网络访问权限；仅在需要时启用bridge/host模式
- **资源限制** —— 从GUI为每个机器人设置CPU和内存上限
- **一键停止** —— 立即终止异常智能体；你的主机始终安全
- **端口冲突检测** —— 在启动前警告主机端口是否已被占用

### 技能管理

浏览50+个内置技能，搜索ClawHub注册表，一键安装/卸载技能 —— 全部在ClawPier内的可视化技能浏览器中完成。
